Demonstration of Positioning

The main content area scrolls normally, allowing the user to view all the content without the side navigation obstructing the main flow.

How Sticky Positioning Works:

The `sticky` element (`.sidebar`) behaves like `position: relative` within its parent (`.container`) until the scroll position meets a defined threshold (`top: 20px` in the CSS). At that point, it switches behavior to mimic `position: fixed`, sticking to the viewport. Crucially, it only sticks within its direct parent element's boundaries.

Contrast with Fixed Positioning:

The **Fixed Box** in the corner uses **`position: fixed`**. This element is removed completely from the normal document flow and is positioned *only* relative to the viewport. It never scrolls with the content, and it doesn't care about its parent container. It stays locked in place at `right: 20px` and `bottom: 20px` indefinitely.
